# Результативность — ВАС (НИО-5)

Веб-приложение для учёта результативности научно-педагогических сотрудников и
формирования ежемесячных отчётов в формате `.xlsx`.

Сотрудник набирает баллы по критериям результативности (задания вышестоящих
органов и руководства ВАС, регистрация ПО в ФИПС, научные статьи, доклады на
конференциях и т.д.), а приложение автоматически считает итог и выгружает
готовый отчёт по шаблону. Начальник подразделения в своём аккаунте видит отчёты
всех сотрудников, проверяет детализацию и приложенные документы, утверждает или
отклоняет отчёты и управляет учётными записями.

## Возможности

- **Подача отчёта** — добавление достижений по категориям с автоматическим
  подсчётом баллов (лимит 30 б.) и выгрузкой `.xlsx`.
- **Реестр приказов** — загрузка оперативных заданий из `.docx` с
  автозаполнением полей.
- **Архив и статистика** — история отчётов сотрудника со статусами
  (на проверке / утверждён / отклонён).
- **Роли**:
  - *Сотрудник* — видит и подаёт только свои отчёты.
  - *Начальник* — контроль и статистика по всем сотрудникам, утверждение /
    отклонение отчётов, управление аккаунтами и архивом.
- **Защищённые аккаунты** — логин и пароль, роли сотрудника, начальника и
  администратора, временные пароли и управление активными сеансами.

## Технологии

- **Backend:** FastAPI + Uvicorn, SQLite
- **Frontend:** vanilla JS (SPA), без сборки
- **Документы:** `python-docx` (разбор `.docx`), `openpyxl` (выгрузка `.xlsx`)

## Установка и запуск

Требуется Python 3.10+.

```bash
pip install -r requirements.txt
python main.py
```

Перед первым входом создайте администратора:

```bash
python manage.py bootstrap-admin --username admin
```

Приложение поднимется на `http://127.0.0.1:8000` и откроется в браузере
автоматически. Базу данных (`data.db`) и папки `uploads/`, `reports/`, `logs/`
приложение создаёт при первом запуске.

На Windows можно использовать пакетные файлы:

```
install.bat   :: установка зависимостей
run.bat       :: запуск сервера
```

> Адрес и порт можно переопределить переменными `HOST` и `PORT`, например
> `set PORT=8000 && python main.py`.

## Структура

```
main.py          — точка входа, все API-эндпоинты
db.py            — схема SQLite, миграции, начальное заполнение
auth.py          — пароли Argon2id, серверные сессии, блокировки и аудит
manage.py        — первоначальное создание администратора и сброс паролей
parsers.py       — разбор .docx (приказы, ПО, статьи)
export.py        — генерация .xlsx по шаблону template.xlsx
software_doc.py  — генерация ведомости ПО
static/          — фронтенд (index.html, styles.css, js/)
template.xlsx    — шаблон итогового отчёта
```

## Примечание

Проект переводится в серверный режим для локальной сети подразделения. Текущая
версия уже использует защищённые аккаунты, но SQLite остаётся переходным
вариантом. До миграции на PostgreSQL сервер следует запускать одним процессом.
Подробности — в `DEPLOYMENT.md`.

База данных и загруженные документы содержат персональные данные и в репозиторий
не включаются (см. `.gitignore`).
